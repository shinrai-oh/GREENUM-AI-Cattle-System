## 1. 引言 (Introduction)





### 1.1 背景与现状



在规模化肉牛养殖中，TMR（全混合日粮）的精准配比至关重要。目前养殖场普遍依靠人工操作和记录，导致配比误差大、数据不透明、缺乏长期数据沉淀，难以进行精细化成本控制和营养分析。



### 1.2 产品目标



开发一套基于计算机视觉的 TMR 饲料配比智能监测系统，核心目标是在**极简硬件成本**下实现智能化监控：

1. **单摄双用：** 单台 TMR 搅拌车仅需安装 1 个高分辨率摄像头，即可同时覆盖“投料口”与“称重显示屏”。
2. **自动化数据采集：** 利用 AI 同时从一个视频流中分离出饲料种类信息和重量信息，替代人工记录。
3. **实时监控与纠偏：** 实时计算当前配比，确保 TMR 质量稳定性。
4. **数据资产化：** 形成饲喂大数据，为精细化养殖提供决策依据。
5. **本地化易部署：** 适应养殖场网络环境，支持 Windows 环境下的本地化闭环运行。

## 3. 产品概述 (Product Overview)



本系统采用“一机双眼”的软件算法架构。物理上，每台 TMR 车仅安装一个广角高分辨率工业相机。软件上，系统将在视频画面中划定两个不同的感兴趣区域（ROI, Region of Interest），并行运行两套不同的 AI 模型。

- **ROI-1（投料区）：** 运行图像分类/物体检测模型，识别饲料种类。
- **ROI-2（仪表区）：** 运行 OCR 模型，识别重量数字。



### 3.1 核心流程图



```
单路摄像头视频流 -> 视频帧抽取 -> [并行处理分支 A: 裁切 ROI-1 (投料口) -> 饲料种类识别模型] + [并行处理分支 B: 裁切 ROI-2 (仪表盘) -> 图像增强 -> OCR 重量识别模型] -> 数据融合 (时间戳对齐) -> 实时配比计算 -> 存入数据库
```



## 4. 功能需求 (Functional Requirements)





### 4.1 核心业务模块





#### 4.1.1 视频采集与双任务实时识别 (Dual-Task AI Core)



- **F1-1 多路视频流并发接入：** 系统需支持 10+ 路视频流接入（每台 TMR 1 路）。
- **F1-2 ROI 区域配置（新增关键功能）：**
  - 初始化时，提供界面让实施人员在画面上分别框选“投料监控区(ROI-1)”和“重量显示屏区(ROI-2)”。
  - 系统需记住该区域坐标，后续自动进行画面裁切分析。
- **F1-3 饲料种类识别 (基于 ROI-1)：**
  - 仅对 ROI-1 区域画面进行分析，识别当前投入的饲料种类（如青贮玉米、苜蓿干草、豆粕等）。
  - *验收标准：* 常见种类识别准确率 > 95%。
- **F1-4 重量数据读取 (基于 ROI-2)：**
  - 仅对 ROI-2 区域画面进行 OCR 分析，识别电子称重屏上的数字。
  - *技术要求升级：* 需支持“小目标 OCR”，或要求硬件具备至少 4K 分辨率以保证数字清晰度。
  - *验收标准：* 重量数值识别准确率 > 98%，能正确识别正负值。



#### 4.1.2 实时配比监控 (Real-time Monitoring)



- **F2-1 TMR 工作站看板：** 展示所有 TMR 车的实时状态、当前配方、已投入原料及误差比例。
- **F2-2 实时告警：** 原料投入量超误差范围（如 ±5%）时，界面高亮报警，可选触发声光报警器。



#### 4.1.3 配方管理 (Formula Management)



- **F3-1 配方录入：** 营养师预录入标准 TMR 配方（原料组成及目标重量/比例）。
- **F3-2 任务下发：** 每日将生产计划下发给指定的 TMR 车。



#### 4.1.4 数据分析与报表 (Data & Reporting)



- **F4-1 自动统计：** 自动生成日、月、年报表（按车、人、牛舍统计计划 vs 实际投料量、准确率等）。
- **F4-2 追溯查询：** 支持查看历史车次的详细投料记录，包含关键时刻的视频截图留证。



### 4.2 系统管理模块



- **F5-1 本地化部署支持：** 提供 Windows 一键安装包，内含所需运行环境。
- **F5-2 设备管理：** 管理摄像头 IP、绑定 TMR 车辆编号。



## 5. 非功能需求 (Non-Functional Requirements)





### 5.1 性能需求



- **分辨率要求：** 摄像头输入分辨率建议不低于 **400万像素 (2K)**，推荐 **800万像素 (4K)**，以满足单画面同时看清大投料口和小仪表盘的需求。
- **并发性与算力：** 支持 10 路以上并发。单路需跑两个模型，对 GPU 要求较高，建议配置 NVIDIA RTX 4070 Ti 或以上级别显卡。
- **实时性：** 从投料到界面更新数据延迟 < 3 秒。
- **稳定性：** 支持 7x24 小时运行，具备自动故障恢复能力（Watchdog）。



### 5.2 环境需求



- **操作系统：** Windows 10 / 11 / Server 2019+ (64-bit)。
- **硬件推荐：** CPU i7+, 内存 32GB+, 1TB SSD, 高性能独立显卡。



## 6. 风险与应对 (Risks & Mitigation)



| **风险点**                  | **描述**                                 | **应对策略**                                                 |
| --------------------------- | ---------------------------------------- | ------------------------------------------------------------ |
| **安装位置受限 (高危)**     | 难以找到兼顾投料口和显示屏的完美安装点。 | 1. 使用变焦或超广角镜头。2. 制定严格的安装支架标准。3. 备选方案：实在无法兼顾时允许加装低成本 USB 摄像头专拍仪表盘。 |
| **分辨率不足导致 OCR 失败** | 广角画面中仪表盘占比太小，数字模糊。     | 1. 强制要求 4K 摄像头。2. 算法引入超分辨率技术放大 ROI-2 区域。 |
| **震动影响**                | TMR 工作时震动大，导致画面模糊。         | 1. 选用带光学/电子防抖的工业相机。2. 算法增加视频稳像预处理。 |

# PART 2: 前端与后端系统设计 (PRD 子文档)





## 1. 系统架构概述



本系统为 B/S 架构，在客户 Windows 机器上实现全本地化部署。

- **前端 (FE):** 现代 Web SPA (Vue/React)，负责交互、可视化、视频播放。
- **后端 (BE):** Windows 服务形式运行，负责业务逻辑、数据持久化、AI 引擎桥接。
- **通信:** HTTP (常规业务), WebSocket (实时推送), RTSP/WebRTC/FLV (视频流)。



## 2. 前端子系统需求 (Frontend)





### 2.1 核心界面模块



- **实时监控看板 (Dashboard):** 卡片式布局展示每台 TMR 状态。包含实时画面缩略图、当前任务进度、目标 vs 实际重量（高亮显示误差）。
- **配方与任务管理:** 配方库增删改查；日历视图拖拽下发每日生产任务。
- **数据报表中心:** 多维度筛选（时间、车辆、人员）；可视化图表（误差趋势图、消耗饼图）；明细追溯（支持查看投料时的**视频截图/回放**）。
- **摄像机标定 (Camera Calibration) - 核心:**
  - 提供直观界面，显示实时画面。
  - 允许用户拖拽绘制**绿色框 (ROI-1 投料区)** 和 **红色框 (ROI-2 仪表区)**。
  - 支持区域旋转微调，保存坐标至后端。



## 3. 后端子系统需求 (Backend)





### 3.1 核心服务模块



- **AI 引擎桥接服务 (AI Bridge):**
  - 接收 AI 推送的结构化数据（含时间戳、ROI-1 种类结果、ROI-2 重量结果）。
  - 数据清洗（过滤低置信度数据）、状态机判断（加料/搅拌/卸料）、实时计算配比误差。
  - 通过 WebSocket 广播实时状态，并将有效投料事件存入数据库。
- **视频流媒体服务:** 集成轻量级流媒体服务（如 MediaMTX, FFmpeg），将 RTSP 转为前端可播放格式（FLV/HLS/WebRTC）。
- **业务逻辑 API:** 提供 RESTful API 支持前端的配方管理、报表查询、ROI 配置存储等功能。



### 3.2 数据存储 (Database)



推荐使用 Mysql 以支持高并发写入。核心表包括：`TMR_Device` (设备信息), `Camera_ROI` (ROI坐标), `Feed_Formula` (配方), `Daily_Task` (日任务), `Feeding_Event` (高频投料明细，含截图路径)。



## 4. 部署与运行环境



- **一键安装包:** 提供 Windows Installer，自动安装数据库、注册后端服务为开机自启、部署前端资源、安装必要 VC++ 运行库。
- **看门狗 (Watchdog):** 守护进程监控 AI 及后端服务状态，崩溃即重启。



## 5. 开发优先级 (Priority)



- **P0:** AI 桥接服务与数据库打通（稳定接收 10 路数据）。
- **P0:** 前端 ROI 标定界面与后端存储接口（V1.1 核心）。
- **P1:** 实时监控看板 WebSocket 联调。
- **P1:** 配方管理基础功能。
- **P2:** 报表系统及可视化。
